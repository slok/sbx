### Task 8: Copy Command

**Goal:** Copy files and directories between host and sandbox.

**Context:** Foundation for file transfer. Uses colon syntax like `scp` and `docker cp`. Direction is inferred from which argument has the `sandbox:` prefix. Recursion is automatic for directories.

**Deliver:**

1. Add `CopyTo` and `CopyFrom` methods to engine interface
2. Docker engine implementation
3. Firecracker engine implementation
4. Fake engine implementation (for tests)
5. `sbx cp` CLI command

**Engine interface additions:**

```
CopyTo(id, srcLocal, dstRemote) error
CopyFrom(id, srcRemote, dstLocal) error
```

**CLI:**

```bash
# Host → Sandbox
sbx cp ./file.txt my-sandbox:/workspace/
sbx cp ./project/ my-sandbox:/workspace/project/
sbx cp config.yaml my-sandbox:/etc/app/config.yaml

# Sandbox → Host
sbx cp my-sandbox:/workspace/output.txt ./
sbx cp my-sandbox:/var/log/ ./logs/

# Using sandbox ID instead of name
sbx cp ./file.txt 01KGB78ZKA:/workspace/
```

**Direction detection:**

Parse arguments for `<name-or-id>:<path>` pattern:
- First arg has colon → CopyFrom (sandbox to host)
- Second arg has colon → CopyTo (host to sandbox)
- Both or neither have colon → error

**Implementations:**

| Engine | CopyTo | CopyFrom |
|--------|--------|----------|
| Docker | `docker cp <src> <container>:<dst>` | `docker cp <container>:<src> <dst>` |
| Firecracker | `scp -r -i <key> <src> root@<ip>:<dst>` | `scp -r -i <key> root@<ip>:<src> <dst>` |
| Fake | Store in memory map | Return from memory map |

**Behavior:**

- Directories copied recursively (auto-inferred, no `-r` flag)
- Preserve file permissions where possible
- Sandbox must be running
- Trailing slash on destination means "copy into this directory"

**Validation:**

- Sandbox must exist
- Sandbox must be running (status = running)
- Source path must exist
- Colon syntax must be valid

**Error messages:**

```bash
sbx cp ./file stopped-sandbox:/workspace/
# Error: sandbox 'stopped-sandbox' is not running

sbx cp ./nonexistent my-sandbox:/workspace/
# Error: source path './nonexistent' does not exist

sbx cp ./file ./other
# Error: invalid syntax, one argument must specify sandbox (e.g., my-sandbox:/path)
```

**Tests required:**

- CopyTo file (host → sandbox)
- CopyTo directory (host → sandbox)
- CopyFrom file (sandbox → host)
- CopyFrom directory (sandbox → host)
- Error: sandbox not running
- Error: source doesn't exist
- Error: invalid colon syntax
- Docker integration tests
- Firecracker integration tests (if CI supports)

**Out of scope:**

- Sync/watch mode (future task)
- `copy:` block in sandbox YAML (future task)
- Glob patterns (`*.txt`)
