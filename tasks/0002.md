### Task 2: List, Status, Stop, Remove Commands

**Goal:** Complete the sandbox lifecycle CLI commands using the fake engine and storage from Task 1.

**Context:** This task is still building CLI scaffolding. The fake engine simulates state transitions without real VMs. We're validating the command flow, state management, and output formatting before adding real virtualization.

**Deliver:**

1. `sbx list` command
2. `sbx status <n>` command
3. `sbx stop <n>` command
4. `sbx start <n>` command (restart stopped sandbox)
5. `sbx rm <n>` command

**CLI:**

```bash
# List all sandboxes
sbx list
sbx list --status running
sbx list --format json

# Get detailed status
sbx status my-sandbox

# Lifecycle
sbx stop my-sandbox
sbx start my-sandbox
sbx rm my-sandbox
sbx rm my-sandbox --force
```

**List output (table, default):**

```
NAME          STATUS    CREATED
my-sandbox    running   5 minutes ago
test-env      stopped   2 hours ago
```

**List output (json):**

```json
[{"name": "my-sandbox", "status": "running", ...}]
```

**Status output:**

```
Name:       my-sandbox
ID:         abc-123
Status:     running
Base:       ubuntu-22.04
VCPUs:      2
Memory:     2048 MB
Created:    2025-01-30 10:00:00
Started:    2025-01-30 10:00:05
```

**Behavior:**

- All commands use the engine interface from Task 1
- Fake engine updates status in storage (no real VMs)
- `stop` on non-running sandbox: error
- `start` on non-stopped sandbox: error
- `rm` on running sandbox: error (unless `--force`)
- `rm --force`: stops then removes
- Status transitions: pending → running → stopped, any → failed

**Tests required:**

- List with filters
- Status output formatting
- Stop/start state transitions
- Remove with and without force
- Error cases (stop already stopped, rm running, etc.)

**Out of scope:** Reconciliation, remote hosts, real Firecracker engine, networking.
