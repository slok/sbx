### Task 9: Port Forward Command

**Goal:** Forward ports from localhost to running sandboxes via SSH tunnels.

**Context:** Enables local access to services running inside sandboxes. Firecracker uses SSH tunnels for dynamic port forwarding. Docker does not support dynamic port forwarding — ports must be defined at container creation.

**Deliver:**

1. Add `Forward` method to engine interface
2. Firecracker implementation (SSH tunnel)
3. Docker implementation (error with helpful message)
4. `sbx forward` CLI command
5. Remove the port-forwarding config from the current config yaml and its implementation models.

**Engine interface addition:**

```
Forward(id, portMappings) error  // Blocks until cancelled
```

**CLI:**

```bash
# Single port (local:remote)
sbx forward my-sandbox 8080:8080

# Short form (same local and remote)
sbx forward my-sandbox 8080

# Multiple ports
sbx forward my-sandbox 8080:8080 3000:3000 5432:5432

# Using sandbox ID
sbx forward 01KGB78ZKA 8080:8080
```

**Output:**

```
Forwarding ports for my-sandbox:
  localhost:8080 → sandbox:8080
  localhost:3000 → sandbox:3000

Press Ctrl+C to stop
```

**Firecracker implementation:**

SSH tunnel using the VM's internal IP:

```bash
ssh -N -L 8080:localhost:8080 -L 3000:localhost:3000 \
    -i ~/.sbx/ssh/id_ed25519 \
    -o StrictHostKeyChecking=no \
    root@<vm-ip>
```

- `-N` — no remote command, just forward
- `-L` — local port forwarding
- Blocks until Ctrl+C or connection drops
- Multiple `-L` flags for multiple ports

**Docker implementation:**

Return error with helpful message:

```
Error: port forwarding not supported for Docker engine.

Docker requires ports to be defined at container creation.
Recreate the sandbox with ports in your sandbox.yaml:

  ports:
    - 8080:8080
    - 3000:3000

Then access directly via localhost:<port>
```

**Port mapping parsing:**

| Input | Local | Remote |
|-------|-------|--------|
| `8080:8080` | 8080 | 8080 |
| `9000:8080` | 9000 | 8080 |
| `8080` | 8080 | 8080 |

**Validation:**

- Sandbox must exist
- Sandbox must be running
- Port format must be valid (`port` or `local:remote`)
- Ports must be in valid range (1-65535)
- Engine must support forwarding (Firecracker yes, Docker no)

**Behavior:**

- Foreground process, blocks until Ctrl+C
- Graceful shutdown on SIGINT/SIGTERM
- Reconnect on SSH connection drop? (optional, stretch)

**Tests required:**

- Port mapping parsing (8080, 8080:8080, 9000:8080)
- Invalid port format errors
- Docker returns helpful error
- Firecracker constructs correct SSH command
- Integration test: forward port, curl localhost, get response

**Out of scope:**

- Background mode (`--background` flag)
- Persistent port definitions in YAML
- UDP forwarding (TCP only)
- iptables-based forwarding (SSH tunnels only)
